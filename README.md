# Say It — Video Pipeline (Remotion starter)

This is a working starter for the "Say It" video pipeline: spine-driven lesson
data in, a fully rendered 4K lesson video out — built so that ad hoc edits
("add a picture at 2:35") can be handled by asking Claude Code directly,
with no separate app in between.

## Why Remotion

- Free for individuals and for-profit teams of 3 or fewer (commercial use
  included). Above that, a company license applies — check remotion.pro
  when it becomes relevant.
- Compositions are React/TypeScript, so timing is exact: frame = seconds × fps.
  That's what makes "add X at time Y" a one-line, deterministic edit instead
  of a request routed through a vendor's JSON schema.
- Self-hosted rendering (local machine or your own AWS Lambda) — no per-render
  vendor fee, only AWS compute if you use Lambda.

## How data flows in

```
A1 Spine / Lesson Scripts workbook
        │
        ▼
  lesson-XX.json            ← auto-generated by your export script
  (pipeline content:            (never hand-edited — regenerate freely)
   segments, timing, text)
        │
        ▼
  lesson-XX-overrides.json  ← hand-edited (by you, via Claude Code)
  (manual overlays only)        (never touched by regeneration)
        │
        ▼
     merge.ts  ──►  LessonVideo.tsx  ──►  remotion render  ──►  .mp4
                                                  │
                                                  ▼
                                     your existing FFmpeg QC pass
                                     (loudness → -14 LUFS, transcribe-and-diff)
```

Keeping generated content and manual overrides in separate files is the
important part: re-running the pipeline after you update the spine never
wipes out a picture you asked to add last week.

## The "just ask" workflow

Say you're previewing Lesson 3 (French) and want a picture of the Eiffel
Tower at 2:35. Open this project in Claude Code and just say so. Claude Code:

1. Converts 2:35 → 155 seconds → frame 4650 (at 30fps): `155 × 30 = 4650`
2. Sources or generates the image (Firefly API, or you drop in a file)
3. Adds one object to `lesson-03-overrides.json`:
   ```json
   { "id": "eiffel-tower", "atSeconds": 155, "durationInSeconds": 4,
     "type": "image", "src": "assets/overlays/eiffel-tower.png",
     "position": "bottom-right" }
   ```
4. Runs `npx remotion studio` so you can preview it in place
5. On approval, runs `npx remotion render` for the final file

No code architecture changes for this kind of request — the overlay layer
in `LessonVideo.tsx` already renders anything in that array. The same
pattern extends to other one-off asks ("swap this word's image," "hold the
scorecard 2 seconds longer," "mute segment 4") — all just edits to a JSON
file or a small, targeted code change, made conversationally.

## Running this project

You'll need Node.js and to run this somewhere with full network access
(Claude Code Desktop, not this sandboxed chat) — Remotion downloads a
headless Chromium on first render.

```bash
npm install
npx remotion studio      # live preview in the browser
npx remotion render src/index.ts Lesson01-FR out/lesson-01-fr.mp4
```

## Scaling to 30 lessons × 5 languages

- Each lesson becomes its own `Composition` in `Root.tsx` (or generate them
  programmatically from a manifest — happy to help wire that up).
- Images are language-agnostic per your existing workflow — only the
  `word.text`, `word.translation`, `word.phonetic`, and audio track change
  per language. The same `LessonVideo` component and JSON shape cover all
  five channels.
- For batch rendering all 150 (lesson × language) combinations in parallel
  rather than one at a time, use Remotion Lambda — cost is typically cents
  per video, no per-render vendor fee.
